# 玩法决策手册

技能的"灵魂"在这里：怎么把"模型讨论 + 实时倍率"变成最终的三档玩法。
核心原则：**讨论为主、倍率校验**——先看采纳模型的讨论方向，再用去水概率找性价比/价值点。

---

## 一、读讨论（产出每场判断）

对**用户勾选的 agent**（默认 Claude + DeepSeek），逐个读 `fan_subjective_prediction_md`，
抽出这几样写进 analysis：
- **方向**：胜/平/负、让球赢盘/输盘、看好谁。优先用结构化 `bet`；`bet=={}`（暂无投注）时从讨论判断倾向。
- **3-5 个核心论点**：原文里"我真正看重的点 / 我不太买账的点"，保留具体细节（球员、区域、数据分位），这是价值所在。
- **最可能比分（双口径对账，别只抄讨论）**：报告每场常驻那行"最可能比分"是**你**填进 analysis 的 `most_likely_scores`。过去只照抄嘉豪讨论里写的那串——但嘉豪偏爱大胜剧本，经验库反复验证"**净胜球普遍被高估**"（见 experience【买法倾向】），照抄就会系统性偏高。所以按三步推，让它经得起市场校验：
  1. **取讨论口径**：各勾选模型在 `fan_subjective_prediction_md` 里写的"最可能比分"。
  2. **拿市场口径校验**：merge.py 已把比分赔率去水算好 `markets.score.top`（去水概率最高的前几个比分）和 `derived.score_implied_1x2`（比分反推的主/平/客）——这是**市场共识**，不是真理，但能照出模型的偏。
  3. **对账定稿**：讨论比分比市场 top 高一个净胜球档时（典型：讨论 3:0、市场 top 是 2:0/1:0）→ 首位放更稳的小胜比分、把大胜降为次选；两口径一致 → 信心更足、可作博比分的依据。`most_likely_scores` 写**对账后**的结果而非原样照抄。报告会把"模型看好"与"市场去水 top"并排显示，差距一眼可见。
- **分歧点**：勾选的多个模型之间哪里一致、哪里冲突。冲突往往就是价值/风险的来源。

未勾选的模型：**只取一句"我更看好"**（merge.py 已抽好 `lean`）做脚注，不读正文。

## 一·补、本届实际表现（赛前真实战况）—— 比分只是结果，表现才是趋势

判断一支队今天怎么踢，**只看它上一场的比分远远不够**。同样是"1:1平"，可能是"压着打、xG领先、中了横梁，就是没运气"（今天该看好），也可能是"被打穿、靠运气偷一分"（今天该警惕）——比分一样、结论相反。所以**凡是今天要分析的队、在本届世界杯已出场过的，都把它上一场（们）的真实战况联网搜出来**，综合成"这支队最近怎么踢的"，再下今天的结论。

> 重要事实：嘉豪预测站只给**终场比分**、不给过程；但这是**真实进行中的世界杯**，过程/战报/数据**联网搜得到**（ESPN/FIFA/赛后报道）。别再误以为"模拟赛、搜不到"——那个旧说法是错的。

### 怎么做
1. **先定该搜哪几场**：`python scripts/fetch_predictions.py history --ids <今天选中的 match_id>`，脚本把这几场两队本届此前已踢的场（对手/比分/当时各模型一句话）列出来；某队"本届首秀"则为空、跳过。
2. **逐场联网搜真实战况**（web-access skill 或可用的联网搜索工具，关键词如 `<队> <对手> World Cup 2026 match report xG possession`）。
3. **抽这几样**（比分看不出、最值钱）：
   - 谁**压着谁打**：控球、射门/射正、**xG**、中楣中框——"该赢没赢"还是"该输没输"；
   - **丢球/进球方式**：定位球？反击？个人失误？点球？——今天对手有没有同款武器；
   - **红黄牌 / 伤停 / 停赛**：直接决定今天**谁不能上**（累黄停赛、伤病）；
   - **关键球员状态、替补奇兵**（如加拿大 Larin 替补 121 秒救平）；
   - 归一句**画像**：①"压着打但低效"（今天利好但带防平味）／②"被压制、靠运气"（今天警惕）／③"真carry碾压"（今天真利好）。

### 怎么用（写进 analysis）
- 综合成每队一两句，写进该场 `form`（字段见第六节），报告显示"本届走势 · 真实战况"块。
- **真实表现 > 比分，也常 > 模型旧讨论**：模型讨论可能没覆盖最新一场——用真实战况**校准**它（讨论说"碾压"、可它上一场其实浪射被平 → 下调净胜球、首选小胜）。
- **重点喂两处**：① `draw_guard`——"**该队本届已被逼平/被偷平**"是平局的**硬先例**信号，比纯结构推断更硬；② `most_likely_scores`——"上一场就吐饼浪射"支持小胜而非大胜。
- **样本极少**（每队本届才 1–2 场）：当**强质化信号**用，别拿一两场做硬统计加权；与当场战术推理冲突时，以推理为主、表现做佐证。
- **fail-open**：联网不可用/搜不到就跳过本步，用现有讨论照常出方案，别阻塞（`form` 留空或标"未联网核实"）。

## 二、倍率校验 + 找价值

把讨论方向和去水概率对照：
- **一致且赔率合理** → 进稳健候选。
- **价值点**：讨论里**明确看好**、但市场去水概率给得**偏低**（赔率偏高）的选项。
  例：DeepSeek/GPT 都说"西班牙难净胜两球"，而"让负(客+2)"市场只给 23% → 这是价值点，写进 `value_points`。
- **陷阱点**：市场很热 / 赔率极低（去水概率很高），但讨论里有人点出明确风险 → 别当稳胆，写进 `trap_points`（报告会以"避雷"块琥珀警示呈现，与价值点的陶土橘形成"机会 vs 风险"对照）。典型如热门强队但勾选模型有人押冷、或"穿盘大胜"看着诱人但两家都说不会大胜。
- 别迷信去水概率，它只是市场共识；模型讨论提供的是市场没充分定价的信息（伤停、战术克制、状态）。

判断"单关 vs 仅过关"：看 `singles` 标记（spf/rqspf/score/goals/htft 各自布尔）。
**只有标记可单关的玩法才能进单关方案**；否则只能进串关。胜平负常常未开放单关（尤其强弱悬殊场）。

## 二·补、防平诊断（Draw-Guard）—— 越是"看着稳"的场，越要防一手

平局是这套系统历史上最贵的教训：第1轮强队共识全平（西0:0/比1:1/沙特1:1/伊朗2:2）、葡萄牙(让-2巨热)被刚果1:1逼平、加波/巴摩/荷日"6家一致却平"。反复证明一件事——**一方看着明显占优、彩民最敢满仓的场，恰恰最容易被一个平局清盘**。所以选完"赢的方向"之后，对这类场要**再单独问一句"这场会不会平？该不该防一手？"**，把结论写进 `draw_guard`，让报告显式提醒用户、由用户自己决定防不防。这一步不是要你改掉主推，而是补一个"看着稳的地方，其实有个平局陷阱"的预警。

### 第一步 · 哪些场要做防平检查（满足任一即查）
- 一方去水胜率 ≥ 60%，或赔率 ≤ 1.3 的超低赔大热门（即用户说的"输赢很明显"）；
- 或平局去水概率 ≥ 28%（势均力敌、本就易平）；
- 别漏了"大热但对手摆大巴/龟缩反击"这种最隐蔽的高危结构。其余场可不查。

### 第二步 · 从数据看平局风险（量化，merge.py 已算好）
- **平局去水概率**（spf 平局的 fair_prob）：≥30% 高危 / 24–29% 警惕 / <22% 低。
- **比分榜里平局比分的位置**（`markets.score.top`）：0:0 / 1:1 / 2:2 是否排进前二、甚至是最高的单一比分——是的话，等于"市场认为最可能的具体结果就是平"，强警报。
- **两口径背离**：`derived.score_implied_1x2` 的 draw 明显高于胜平负盘口给的平局概率 → 比分市场比胜平负盘更看好平，可疑。
- **结构性高危**：赔率 ≤1.2 的大热 + 对手是摆大巴/龟缩反击型 = "闷平/被偷平"高危（葡萄牙 vs 刚果、加拿大 vs 卡塔尔同款结构）。低赔大热不是"稳"，是"赢了赚得少、平了全报废"的不对称风险。
- **本届被平先例**（来自第 1b 步 `form`/`history`，真实战况）：热门队**本届此前已被逼平/被偷平过**，或它的真实表现是"压着打但低效"（xG 领先却没赢）→ 平局不是假设、是**已发生过的先例**，权重高于纯结构推断（如加拿大首轮 xG 1.25-0.98 压着波黑却 1:1、卡塔尔刚 1:1 偷平瑞士）。

### 第三步 · 从嘉豪的话看平局风险（质化——数据看不出、最值钱）
- **"最容易打脸我的地方"段落聚类**：把每家这一段拎出来对比。**若 ≥2 家把"被偷平/闷平/对方门将爆发/我方吐饼浪射"写进打脸段、且指向同一剧本 → 这是主线、不是尾部风险。** 葡萄牙 1:1 就是各家打脸段集体预言"门将+定位球偷鸡"、结果应验——这条铁律是用钱换来的。
- **剧本里出现"扳平"情节**：模型最终结论看好赢，但它的"比赛镜头/主观脚本"里演出了对方扳平（如 GLM 写"第68分钟阿菲夫扳平1:1"）→ 说明模型自己也认为对方能进球、零封难，平局的种子已经埋下。读 GLM 尤其要看它剧本里的冷情节，别只看它最后报的比分。
- **有模型直接押平/不败/冷门方向**：哪怕是反指标模型（Gemini 常押冷），它押平本身就是个加权信号——尤其当它点的具体理由（某门将、某软肋、某红牌点）和别家的打脸段重合时，分量加倍。
- **高频"防平词"**：嘉豪反复出现"久攻不下""得势不得分""快乐足球/吐饼/浪射""龟缩死守/低位防守""保平心态""偷一个""闷平/僵局"——这些词扎堆就是警报，别只盯着"碾压""血崩点"那些看好的话。
- **门将是平局剧本的第一主角**：对方门将"扑救评分高/可能爆种" → 易闷平；己方门将差 + 对手有反击或定位球 → 易被偷平。

### 第四步 · 防一手怎么落地（默认只提示 + 给对冲，不动主推）
- 命中的信号写进该场 `draw_guard`（字段见第六节），报告会单独显示一个"防一手·平局"提示块。
- 给**可选**的对冲思路，让用户自己权衡，**不强行改三档主推**：
  - **平局本身有价值就直接点出来**：当平局去水概率不低、赔率可观（如平30%、赔2.91），明说"这场与其猜谁赢，不如顺势押平/做双选"——顺水推舟，不是硬猜冷门。
  - **大热高危场**：提示"只锁胜别锁盘、别博比分；别把它堆进多腿串（一旦平，整串全崩）；想保险可单独配一注平局对冲"。
  - **不主动删主推的腿**（尊重用户判断），但在该场 `draw_guard.hedge` 或 `meta.risk_note` 里点明"这条是整套里最该防平的腿"。
- 记住量级：**串里每多一条"看着稳但有防平警报"的腿，整串崩盘概率就乘一次**。防平的第一性原理是"能单则单、看着越稳越别贪串"。

## 三、三档方案

每次都给三档，让用户临场自己挑。

### 稳健（calm）— 求命中率
- 只取**勾选模型方向一致**、且赔率不过低也不过高的**单关**。
- 宁缺毋滥：某场有输盘分歧、或胜平负未开单关，就**跳过该场**，别硬凑。
- 每腿给信心星级（0-5）和一句理由。

### 平衡（mid）— 命中与回报兼顾
- 一组**核心单关**（稳健底仓）+ **1 个 2-3 腿小串**。
- 小串每腿都要高把握；避免把两个"博"的腿绑一起（串关命中率是各腿相乘，越串越低）。

### 激进（bold）— 博高回报
- **跨场串关**博高赔（可多组），顺着讨论里的"博冷剧本"（如险胜输盘、爆冷、大比分）。
- 加少量**高赔单关**（讨论支持的比分/总进球/半全场）。
- 明说这是低命中、高回报，容忍踢飞。

### 串关赔率
合计赔率 = 各腿赔率连乘（build_report.py 会自动算）。串关任一腿不中则整串作废。

---

## 四、信心星级口径（写 confidence: 0-5）
- 5：勾选模型方向一致 + 去水概率高 + 无明显风险点。
- 4：方向一致，小瑕疵或赔率略低。
- 3：方向基本一致但有分歧/风险，或博中等赔。
- 2：跟某一个模型的"博"判断（如险胜输盘、爆冷）。
- 1：纯博冷/高赔比分。

---

## 五、底线（务必遵守）
- **不报真实下注金额**，只给信心星级 + 相对仓位语言（底仓/小注/博一点）。
- 报告底部固定"理性投注"提示（build_report.py 已内置 disclaimer，可按需覆盖）。
- 不承诺胜率、不打包票；明确"模型分析 ≠ 投注建议"。
- 数据有缺口（某场无倍率、未开售）时如实标注，不脑补。

- 留空的部分可省略；缺 analysis 条目的比赛报告会以"仅数据"呈现。

## 七、retro.json 结构（第 R 步 Track 2 产出 → 喂给 build_report.py --retro）

模型据「上期 analysis.json + retro_facts.json（脚本算的终场/对错）+ 用户实际所买」写成。事实字段抄 `retro_facts.json`，判断字段你写。

```json
{
  "reviewed_run": "2026-06-15", "reviewed_at": "2026-06-16",
  "user_bought": { "tier": "平衡", "legs": ["比利时 胜(单关)"], "note": "或：没买/只看" },
  "user_result": "你买的那注中没中的一句话总结",
  "matches": [
    { "match_id": "54329956", "teams": "比利时 vs 埃及",
      "final_score": "1:1", "result_side": "平局",                       // 脚本(retro.py score)
      "graded": [                                                        // 你判：上期每条推荐对账
        {"play": "比利时 胜(单关)", "from": "稳健", "hit": false},
        {"play": "让胜(比-1) 已避开", "from": "陷阱点", "would_hit": false, "warned_correctly": true} ],
      "model_take": "这场的一句复盘（谁对谁错、为什么）" }
  ],
  "plans_review": {
    "steady":   {"legs": 1, "hit": 0, "verdict": "..."},                // legs/hit 可省；verdict 必有
    "balanced": {"verdict": "..."}, "aggressive": {"verdict": "..."} },
  "model_calibration": [ {"brand": "DeepSeek", "direction_right": "2/3", "note": "更可信在哪"} ],
  "jiahao_signal_lessons": ["被验证可信的信号1", "..."],
  "big_direction_verdict": "大方向对不对一段话",
  "next_adjustments": ["下次买法怎么调1", "..."],
  "historical_synced": 15                                               // 可选：本次 Track 1 新分析的历史场数
}
```
- `graded[].hit`=命中；`warned_correctly`=陷阱点正确避开（渲染成 ⊘ 琥珀）；其余渲染成 ✗。
- **半全场/半场等脚本判不了的腿**：`hit` 必须来自**用户确认**（见 SKILL.md 第 R 步 Track 2 "判定分两层"），不要写 null 猜、也不要"存疑"含糊；确实还没结算才用 `null` 并在 `note` 写"待结算"。
- 渲染位置：三档方案**下方**的"上期复盘回顾"可折叠模块。
- 这些判断同时增量沉淀进 `references/experience.local.md`（本机累积大脑；与共享基线 `experience.seed.md` 一起读，详见 SKILL.md 第 R 步）。
